# **[Week 7] Greedy Algorithm, Dynamic Programming**


## 그리디 알고리즘(탐욕법, Greedy Algorithm)이란?
- "매 선택에서 **지금 이 순간 당장 최적인 답**을 선택하여 적합한 결과를 도출하자"라는 모토를 가지는 알고리즘 설계 기법
- 단, 그리디 알고리즘을 사용하면 매 선택이 그 순간에 대해서는 최적이지만 그걸 종합적으로 봤을 땐 최적이라는 보장은 절대 없다는 것을 명심
- 그리디 알고리즘은 부분의 최적해의 집합이 곧 전체 문제의 해답이 될 때 사용할 수 있음

## 그리디 알고리즘의 특징
그리디 알고리즘은 아래의 특성을 가지는 문제들을 해결하는 데 강점이 있음
- **탐욕 선택 속성(greedy choice property)** : 한번의 선택이 다음 선택에는 전혀 무관한 값이어야 함
- **최적 부분 구조(optimal subtructure)** : 매 순간의 최적해가 문제에 대한 최적해여야 함

## 그리디 알고리즘 문제 해결 방법
- 선택 절차(Selection Procedure) : 현재 상태에서의 최적의 해답을 선택함
- 적절성 검사(Feasibility Check) : 선택된 해가 문제의 조건을 만족하는지 검사함
- 해답 검사(Solution Check) : 원래의 문제가 해결되었는지 검사하고, 해결되지 않았다면 선택 절차로 돌아가 위의 과정을 반복함

## 근사 알고리즘 (approximation algorithm)
- 어떤 최적화 문제에 대한 해의 근사값을 구하는 알고리즘
- 가장 최적화되는 답을 구할 수는 없지만, 비교적 빠른 시간에 계산이 가능하며 어느 정도 보장된 근사해를 계산할 수 있음

## 매트로이드 (Matroid)
- 그리디 알고리즘이 최적해를 가려낼 수 있는 경우인지를 판별할 수 있게 하며, 독립성이라는 성질을 만족하는 수학적 공간
- **매트로이드의 성질을 만족하면 그리디 알고리즘으로 항상 최적해를 찾을 수 있음**

## 동적 계획법(Dynamic Programming)이란?
- 특정 범위까지의 값을 구하기 위해서 그것과 다른 범위까지의 값을 이용하여 효율적으로 값을 구하는 알고리즘 설계 기법
- 어떤 문제를 풀기 위해 그 문제를 더 작은 문제의 연장선으로 생각하고, 과거에 구한 해를 활용하는 방식


## 동적 계획법의 대표적인 방식
1. **`Top-down`** : 큰 문제를 작은 문제로 쪼개어 나가면서 문제를 해결하는 방식, 주로 **재귀** 함수를 사용해 문제를 해결
2. **`Bottom-up`** : **점화식**이라는 전체 알고리즘을 파악하여 작은 문제로 큰 문제를 해결하는 방식

## 메모이제이션(Memoization)
- 동일한 계산을 반복해야 할 경우 **한 번 계산한 결과를 메모리에 저장**해 두었다가 꺼내씀으로써 중복 계산을 방지할 수 있게 하는 기법
- 동적 계획법의 핵심이 되는 기술로써 결국 **메모리라는 공간 비용을 투입해 계산에 소요되는 시간 비용을 줄이는 방식**
